# 图形推理题 - 产品文档

## 基础规则

### 位置排列移动 (Position Permutation)
- 每道题包含 3 行，每行有 3 个田字格（2×2 网格，4个位置：左上TL、右上TR、左下BL、右下BR）
- 每行使用 4 个不同的 emoji，不同行之间的 emoji 互不重复
- 核心规律：**所有** emoji 按照固定的 4-cycle 位置置换规则在 3 个格子间移动
  - Grid2 = P(Grid1)，Grid3 = P(Grid2)，其中 P 是同一个位置置换（6 种可能中随机选一种）
- 3 行共享同一个置换规则，但每行的 emoji 初始排列随机
- 第 3 行的第 3 个格子为待求答案，由玩家从选项中选取填入

## 视觉变换规则（加难机制）

每道题随机选取 **1~2 个**不同的视觉变换规则，分别应用到第 3 行 Grid 3 的**不同位置**上。

被应用变换规则的位置行为：
- Grid 1 和 Grid 2 中的 emoji 正常显示（无变换）
- **仅在第 3 行的 Grid 3 中**，该位置的 emoji 外观发生变化

变换规则对所有行生效，但仅在第 3 行的 Grid 3 中可见。前两行作为示例，帮助玩家发现变换规律。

### 规则 1：缩小 (Shrink)
- Grid 1、Grid 2 中 emoji 正常大小
- Grid 3 中该位置的 emoji 缩小至约 60%

### 规则 2：上下镜像 (Mirror-X)
- Grid 1、Grid 2 中 emoji 正常显示
- Grid 3 中该位置的 emoji 垂直翻转（上下镜像）

### 规则 3：顺时针旋转 90° (Rotate-90)
- Grid 1、Grid 2 中 emoji 正常显示
- Grid 3 中该位置的 emoji 顺时针旋转 90 度

### 规则 4：左右镜像 (Mirror-Y)
- Grid 1、Grid 2 中 emoji 正常显示
- Grid 3 中该位置的 emoji 水平翻转（左右镜像）

## 规则组合方式

每道题随机生成：
1. **一个 4-cycle 位置置换**（6 种中随机选一种，所有 emoji 统一使用）
2. **1~2 个视觉变换规则**（从 4 种中随机选取，不重复），分别绑定到第 3 行 Grid 3 的不同位置上

视觉变换规则的位置分配原则：
- 变换规则应用于**位置**而非特定 emoji，即无论哪个 emoji 移动到该位置，都会呈现对应的变换效果
- 系统会自动排除不适合某些变换的 emoji（如左右对称的 emoji 不应用于左右镜像规则），确保变换后视觉差异明显

玩家需要同时观察：
- emoji 的移动规律（位置置换）
- 哪些**位置**在 Grid 3 中发生了视觉变化（变换规则）

## 答案选项设计

每道题提供 **8 个选项**，来自第 3 行的 4 个 emoji，每个 emoji 出现两次：

### 正确版本（4 个）
每个位置对应一个正确选项，呈现该 emoji 在 Grid 3 中应有的视觉状态：
- 该位置有变换规则 → 显示变换后的样子（缩小/镜像/旋转）
- 该位置无变换规则 → 显示正常样子

### 干扰版本（4 个）
每个 emoji 对应一个干扰选项，呈现错误的视觉状态：
- 该位置有变换规则 → 干扰项显示正常（未变换）的样子
- 该位置无变换规则 → 干扰项显示随机变换后的样子

选项顺序随机打乱，玩家需要同时判断：
- 哪个 emoji 应该放在哪个位置（位置置换规律）
- 该位置应该显示哪种视觉状态（变换规则）

## Emoji 选择与兼容性

### Emoji 池
- 总共 48 个 emoji，涵盖动物、水果、乐器、交通工具、食物等类别
- 优先选择有明确方向性的 emoji（如动物头部、乐器、车辆等）

### 变换兼容性过滤
为确保变换后视觉差异明显，系统会对部分 emoji 进行变换排除：

- **上下镜像 (Mirror-X)** 排除上下对称的 emoji，如：蓝色书本、绿色书本 等
- **左右镜像 (Mirror-Y)** 排除左右对称的 emoji，如：🍎🐯🌻🍔 等
- **旋转 90° (Rotate-90)** 排除旋转对称的 emoji，如：🍎🍏🌸🌻 等
- **缩小 (Shrink)** 无排除项，缩放对所有 emoji 都产生明显视觉差异

### 动态分配机制
- 系统先计算第 3 行 Grid 3 的 emoji 排列
- 然后基于该位置的 emoji 选择兼容的变换规则
- 若找不到足够的兼容组合，则重新生成题目

## 倒计时功能
- 每道题有 **60 秒** 时间限制
- 玩家需要在规定时间内完成所有题目
- 超出时间限制，系统会自动提交当前进度
- 倒计时采用进度条设计，进度条 thumb 在进度走到 45 秒之前是小乌龟，45 秒后变成小兔子
